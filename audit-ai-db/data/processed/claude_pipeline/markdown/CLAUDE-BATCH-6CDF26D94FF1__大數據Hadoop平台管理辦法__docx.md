# 大數據Hadoop平台管理辦法



國泰人壽保險公司大數據Hadoop平台管理辦法

113年1月30日修訂

權責單位：數據暨人工智慧發展部

目的

為規範本公司「大數據Hadoop平台」之數據儲存、處理、程式開發與部署之相關程序，並有效管理本平台相關使用及維運作業，特訂定本辦法。

適用範圍

本公司應依本辦法之規定辦理大數據Hadoop平台相關事宜。

本辦法如有未盡事宜，悉依相關法令、「國泰金融控股股份有限公司暨各子公司大數據Hadoop平台管理要點」（下稱金控大數據平台管理要點）、「國泰人壽保險公司機器學習平台使用管理要點」或金控及本公司其他有關規章之規定辦理。

大數據Hadoop平台

所稱「大數據Hadoop平台」（下稱本平台）係指本公司提供之伺服器環境，其彙整儲存內外部資料做為資料分析建模及提供巨量資料的儲存和處理能力。

前項伺服器環境包含大數據分析平台Hadoop Analysis Platform、大數據資料湖Hadoop DataLake、相關運行伺服器及儲存空間等，或經申請後使用本平台相關之資源，及本平台另外產生的複合式環境。

用詞定義及單位權責

本辦法相關用詞定義及單位權責說明如下：

平台資訊管理單位: 由數據工程組負責本平台之作業流程規劃、資料流系統架構規劃及帳號管理，並擔任內稽內控窗口，下稱作業單位。

平台業務需求單位:係指基於業務需求，經申請且被授權於本平台上進行資料分析或使用之業務單位，下稱需求單位。

有關本辦法之適用，如有未盡事宜，作業單位保有最終解釋權。

平台使用者帳號申請與管理

需求單位人員得因業務需要，依金控大數據平台管理要點檢附相關文件，經需求單位科級主管(或代理人)核准後，向作業單位提出申請或刪除大數據分析平台業務使用者帳號及權限。

作業單位應遵循金控大數據平台管理要點程序，定期檢核各需求單位業務使用者名單及權限。

需求單位使用者如有離職或調動情事者，應於正式生效前，依本辦法第五條，向作業單位申請刪除該員之帳號及權限，並納入內部稽核項目定期查核。

平台電腦作業及程式開發

需求單位因相關業務有電腦作業需求者，應依「電腦作業申請管理要點」程序並檢附相關文件，向作業單位提出電腦作業申請。

作業單位須依「電腦作業申請管理要點」相關程序與需求單位進行需求溝通及程式測試驗收。於驗收完成後，始可依「程式異動管理施行細則」進行正式環境程式部署及上架等作業。

作業單位發現電腦作業發生問題而無法自行排除者，須依「電腦作業問題管理辦法」，逕至問題管理系統填報，或向電腦服務台人員申報。

資料表申請與管理

需求單位申請本平台資料表權限、導入或相關作業，應遵循「數據平台使用管理要點」之規定，透過表單管理系統提出申請，並完整說明使用目的、申請資料表名稱等資訊。本平台資料表權限清查應依「數據平台使用管理要點」定期檢視，以移除無使用需求之項目表格權限。

平台安全管理

本平台須使用VDI平台連線使用。

VDI平台應側錄使用者帳號及作業時間操作畫面，並依「日誌留存暨管理系統管理要點」及「VDI平台管理施行細則」留存側錄主機，以供查核。

本平台相關作業應符合金控大數據平台管理要點及相關管理維護單位之資訊安全政策。

保密義務

本平台之維護人員、開發人員、使用人員，於接觸或使用該資料時，不得有洩漏或移作他用之情事，並需依相關規定使用本平台數據資料。前述人員所屬單位之科級主管負有控管該資料之責任，並應責成相關人員嚴守保密義務。

罰則

如有違反本辦法之規定事項，致影響公司或保戶權益，經查證屬實者，除移送人事管理單位議處外，行為人並應自負相關民、刑事責任。

施行日期

本辦法呈總經理核定後施行，修正或廢止時亦同。
本辦法訂於111年10月03日，歷次修正紀錄如下：112年12月18日、113年1月30日。
