第一條 (目的) 
為規範本公司資訊作業申請程序，加強資料管理及提昇資訊作業品質，特訂定「國
泰金融控股股份有限公司資訊作業申請管理要點」（以下稱本要點）。 
 
第二條 (適用範圍) 
本公司暨子公司（以下簡稱「各公司」）之資訊作業申請，應依本要點之規定辦
理，委外開發、管理維護之資訊系統，如系統之業務管理單位另有規定者，則從
其規定辦理。 
 
第三條 (名詞定義) 
本要點名詞定義如下： 
一、資訊作業：包括但不限於應用系統之開發或修改、作業系統之設定、報表或
檔案之產出等。 
二、儲存媒體：係指記錄資料所用之媒介，包括但不限於硬碟、磁帶、光碟、隨
身碟、紙張和報表等。 
三、機密資料：係指未經公開之財務資訊、客戶資料、股東資料、員工資料或其
他經公司列為機密之資料等。 
四、子公司：係指依金融控股公司法規定，本公司所屬之子公司。 
五、外部單位：係指各公司以外之自然人、法人或非法人體。 
六、承辦單位：係指各公司資訊單位。 
七、申請單位：係指提出資訊作業申請之單位。 
八、業務管理單位：係指各公司特定業務管理之權責單位。 
九、特定業務系統：係指業務管理單位為其特定業務而設置 之系統；包含但不
限於金控人事系統、金控風險管理系統、金控稽核系統等。 
 
第四條 (管理單位及權責) 
承辦單位應負責之事項如下： 
一、負責受理、覆核各申請單位之資訊作業申請。

第五條  
業務管理單位應負責之事項如下： 
一、 負責審核資訊作業申請。 
二、 確認申請書內容是否符合規定。 
 
第六條  
申請單位應負責之事項如下： 
一、 提出資訊作業申請，並完整說明需求。 
二、 於資訊作業完成後，進行測試、驗收。 
 
第七條 (作業分類) 
本要點之資訊作業申請項目共分為下列三類： 
一、 跨公司客戶資料分析作業申請。 
二、 特定業務系統資訊作業申請。 
三、一般資訊作業申請。 
 
第八條 (跨公司客戶資料分析作業申請) 
跨公司客戶資料分析作業之業務管理單位為數位數據暨科技發展中心數據科技
發展部。 
辦理跨公司客戶資料分析作業，須依本要點提出申請，經數位數據暨科技發展中
心數據科技發展部依法審核，並會簽相關子公司；其申請流程如附件一「跨公司
客戶資料分析作業申請流程」所示。 
 
第九條 (特定業務系統資訊作業申請) 
辦理特定業務系統資訊作業須依附件二「特定業務系統資訊作業申請流程」所示
流程提出申請，經業務管理單位審核後，由承辦單位配合辦理之，但該申請之特
定業務另有規定者，從其規定。    
 
第十條 (一般資訊作業申請) 
辦理一般資訊作業，須依附件三「一般資訊作業申請流程」所示流程提出申請。 
 
第十一條 (申請應遵循事項) 
資訊作業申請應遵循事項如下： 
一、申請單位應透過「金控資訊作業申請系統」（以下簡稱「申請系統」）提出
申請。

二、申請單位提出資訊作業申請前，應與承辦單位先行聯繫協調，以確保作業規
劃妥適並兼顧完成時效。 
三、如資訊作業可能影響其它單位時，應先與該單位協調，並檢附協 
    調紀錄、往返郵件或會簽簽呈等。 
四、資訊作業申請書應檢具作業所需之相關附件。 
五、申請單位除本管理要點外，另應遵循各業務管理單位訂定之其他管理規定。 
 
第十二條 (作業應遵循事項) 
承辦單位作業程序應遵循事項如下： 
一、承辦單位受理資訊作業申請後，應指派專人(以下簡稱「承辦人員」)負責該
項作業。 
二、承辦人員應與申請單位人員進行需求內容分析，瞭解細部需求，並依需求進
行作業。 
三、承辦單位完成作業，並初步測試無誤後，應至申請系統，點選作業完成，轉
由申請單位進行測試、驗收。 
 
第十三條 (測試驗收應遵循事項) 
申請單位進行測試、驗收之程序及應遵循事項如下： 
一、資訊作業測試時，申請單位應提供測試資料；如受客觀條件限制無法提供時，
得由承辦單位協助準備測試資料。 
二、若測試、驗收不符需求，應立即向承辦人員反應，由承辦人員進行修正，修
正後申請單位應再重新進行測試、驗收。 
三、申請單位測試、驗收無誤後，由申請單位人員至申請系統中，點選驗收完成，
將該申請書結案歸檔。 
 
第十四條 (資料管理) 
申請單位所申請之資訊作業，如涉及特定業務資料之使用時，應事先取得該業務
管理單位之同意。 
 
第十五條 (資料控管) 
如因業務需要，須將資料轉錄至其它儲存媒體或轉送至外部單位者，應遵循事項
如下： 
一、進入申請系統向承辦單位提出申請，並取得業務管理單位同意。 
二、機密資料之轉錄或轉送，應考量予以適當遮蔽或加密，並應遵守相關法令及
規章之規定。 
 三、資料轉送至外部單位時，需遵守「國泰金融控股股份有限公司資訊傳輸管
理要點」或各公司相關規定辦理。
